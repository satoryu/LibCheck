# 蔵書検索結果の在庫状況順ソート — Requirements

## Problem Statement

ISBN 蔵書検索結果画面 (`BookSearchResultPage`) の蔵書状況リストは、登録図書館の**登録順**でそのまま表示されている。利用者は「借りられる図書館」を探すのにリスト全体を確認する必要がある。

「貸出可能・蔵書あり」の図書館を上位に、「蔵書なし」等を下位に並べ替え、借りられる館をすぐ見つけられるようにする。

## Requirements

### Functional
- FR-1: 蔵書状況リストを在庫状況の優先度降順で表示する（貸出可能・蔵書あり が最上位）。
- FR-2: 同じ優先度の図書館同士は、従来どおり登録順を保持する（安定ソート）。

### Non-Functional
- NFR-1: 既存の優先度定義 `availabilityPriority` を再利用し、新たな優先度ルールを作らない。
- NFR-2: 表示順のみの変更とし、検索履歴の保存内容・API 呼び出しには影響を与えない。
- NFR-3: React Query キャッシュ（登録図書館配列）をミューテートしない。
- NFR-4: Clean Architecture の依存方向を維持（presentation→domain のみ）。

## Constraints
- `'貸出可'` と `'蔵書あり'` は同一の `AvailabilityStatus.available` にマップされる（既存仕様）。よって両者は同列で最上位。
- 並べ替え対象は結果が得られた登録図書館のみ。result に存在しない館は最下位相当（unknown）として扱い、例外を出さない。

## Acceptance Criteria
- AC-1: 登録順が「蔵書なし→貸出可能」でも、結果画面では貸出可能の館が上に表示される。
- AC-2: 優先度が同じ図書館は登録順のまま。
- AC-3: 既存のテストと型チェック（`npx tsc -b`）がすべて通る。

## User Stories
- US-1: 利用者として、検索結果で借りられる図書館がリスト上部にまとまっていてほしい。すぐに来館先を判断できる。
