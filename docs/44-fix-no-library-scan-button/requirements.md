# Issue #44: 図書館未登録時の検索結果ページに不適切な「別の本をスキャンする」ボタンが表示される

## Problem Statement

図書館が未登録の状態で書籍を検索した場合、検索結果ページに「別の本をスキャンする」ボタンが表示される。しかし、図書館が未登録の状態では別の本をスキャンしても同じ結果になるため不適切。

## Requirements

### Functional Requirements

- 図書館未登録状態では「別の本をスキャンする」ボタンの代わりに「図書館を登録する」ボタンを表示する
- 「図書館を登録する」ボタンをタップすると `/library/add` ページに遷移する

### Non-Functional Requirements

- 既存の他の状態（検索結果表示、エラー状態）のボタンは変更しない

## Constraints

- `go_router` の `context.push()` を使って遷移する

## Acceptance Criteria

1. 図書館未登録時に「別の本をスキャンする」ボタンが表示されないこと
2. 図書館未登録時に「図書館を登録する」ボタンが表示されること
3. 「図書館を登録する」ボタンタップで `/library/add` に遷移すること
4. 既存のテストがすべてパスすること

## User Stories

- ユーザーとして、図書館が未登録の場合に図書館登録ページに直接遷移できるボタンがあることで、スムーズに設定を完了したい
