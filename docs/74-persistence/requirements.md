# 登録図書館・検索履歴のサーバー永続化（#74 / Cloudflare Phase 3） — Requirements

関連: 移設 #78 の Phase 3。前段 = #73 認証（Google/GIS）。基盤 = Cloudflare Pages Functions（#79）。

## Problem Statement

現在 `localStorage` に保存している **登録図書館** と **検索履歴** を、認証済みユーザーに紐づけて **Cloudflare D1** に永続化し、端末をまたいで同じデータを使えるようにする。

## 採用方針（意思決定済み）
- **必須ログイン化**：未ログインでは全機能を使えない（ゲスト廃止）。ログイン前は専用ログイン画面を表示。
- **セッション維持**：GIS の自動選択（One Tap）で起動時にサイレント復帰を試みる（トークンは保存しない）。失敗時はログイン画面。
- **データ移行・サーバー/ローカル切替は無し**：登録図書館・検索履歴は常にサーバー（D1）。localStorage 版はこれらの用途から外す。
- **検索履歴もサーバー化**：登録図書館と同じパターンで D1 に永続化。

## Requirements

### Functional
- FR-1: ログイン必須。未ログインはログイン画面のみ表示し、ログイン後にアプリ本体を表示する。
- FR-2: 起動時に GIS 自動選択でログイン状態の復帰を試みる（できなければログイン画面）。
- FR-3: 登録図書館をユーザー単位で D1 に保存・取得・追加・削除できる（既存 `RegisteredLibraryRepository` の挙動を維持）。
- FR-4: 検索履歴をユーザー単位で D1 に保存・取得・削除・全削除できる（既存 `SearchHistoryRepository` の挙動を維持）。
- FR-5: 別端末で同じユーザーでログインすると、同じ登録図書館・検索履歴が見える。

### Non-Functional
- NFR-1: 保護 API は `Authorization: Bearer <ID token>` を `jose` で検証（`iss`/`aud`/`exp`、Google JWKS）。検証は #73 の `verifyGoogleIdToken` を `functions/_shared/` に共有化して再利用。
- NFR-2: Clean Architecture 維持。既存リポジトリ・インターフェース（getAll/add/remove 等）は変えず、実装をサーバー版に差し替える。呼び出し側（フック）の改変は最小。
- NFR-3: シークレット非混入。Google クライアント ID は公開値。D1 バインディングはサーバー側のみ。
- NFR-4: Vitest / 型チェック維持・グリーン。サーバー検証・リポジトリはテスト可能に。

## Constraints
- 実装は Cloudflare Pages Functions（Workers ランタイム）+ D1（SQLite）。
- ID トークンは短命（約1時間）。期限切れ後の保護 API 401 はログアウト扱い（再ログイン）とする（高度な更新は本 Phase 外）。
- D1 作成・バインド・スキーマ適用・Pages env `GOOGLE_CLIENT_ID` 設定には Cloudflare 認証が必要（ユーザー手元のトークンで代行）。

## Acceptance Criteria
- AC-1: 未ログインでアプリを開くとログイン画面が出る。ログインするとアプリが使える。
- AC-2: 登録図書館の追加・削除がサーバー（D1）に反映され、リロード/別端末でも保持される。
- AC-3: 検索を行うと検索履歴が D1 に保存され、履歴画面・別端末で見える。履歴の個別削除・全削除ができる。
- AC-4: 保護 API は有効トークンで 200、無効/欠如で 401（検証ロジックを単体テストで担保）。
- AC-5: 型チェック・全テストがグリーン。シークレット/実値がリポジトリに含まれない。

## User Stories
- US-1: 利用者として、ログインすれば別の端末でも自分の登録図書館と検索履歴をそのまま使いたい。
- US-2: 利用者として、毎回のリロードで再ログインさせられたくない（自動復帰）。
