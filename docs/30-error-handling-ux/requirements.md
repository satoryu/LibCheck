# Issue #30: エラーハンドリング・UX改善 — 要件定義

## Problem Statement

現状のエラーハンドリングにはいくつかの問題がある:

1. **生のエラーオブジェクトがユーザーに表示される**: `LibraryManagementPage` で `Text('エラー: $error')` として技術的なエラーメッセージがそのまま表示される
2. **エラー表示が画面ごとに不統一**: 一部のページはアイコン+メッセージ+リトライボタン、一部はテキストのみ
3. **ローディング表示が不統一**: 一部のページは補助テキスト付き、一部は `CircularProgressIndicator` のみ
4. **例外の種類に応じたメッセージ分岐がない**: ネットワークエラーもタイムアウトも同じ「エラーが発生しました」で表示される

## Requirements

### Functional Requirements

1. **共通エラー表示ウィジェット**: 再利用可能な `ErrorStateWidget` を作成し、全画面で統一したエラー表示を行う
2. **ユーザーフレンドリーなエラーメッセージ**: 例外の種類（ネットワーク・HTTP・パース・タイムアウト）に応じて分かりやすいメッセージを表示する
3. **リトライ機能**: すべてのエラー状態にリトライボタンを表示する
4. **ローディング表示の統一**: すべてのローディング状態にコンテキストに応じた補助テキストを表示する
5. **既存ページの修正**: `LibraryManagementPage`、`LibraryListPage`、`CitySelectionPage`、`BookSearchResultPage`、`SearchHistoryPage` のエラー/ローディング表示を統一する

### Non-Functional Requirements

1. **一貫性**: 全画面で同じパターンのエラー/ローディング表示を使用する
2. **テスタビリティ**: エラーメッセージのマッピングがテスト可能であること

## Constraints

- 既存の `CalilApiException` 階層を活用する
- デザインガイドラインのエラーカラー（`#D32F2F`）に従う

## Acceptance Criteria

1. 全画面のエラー表示が `ErrorStateWidget` を使用して統一されている
2. `CalilNetworkException` / `CalilTimeoutException` などに応じた日本語メッセージが表示される
3. 生のエラーオブジェクトがユーザーに見える箇所がない
4. 全画面のローディング表示に補助テキストが付いている
5. すべての修正箇所にテストが存在する

## User Stories

1. ネットワーク不通時に「インターネット接続を確認してください」と表示され、リトライボタンで再試行できる
2. APIタイムアウト時に「応答に時間がかかっています。再度お試しください」と表示される
3. ローディング中に「読み込み中...」等の補助テキストが表示され、操作待ちであることが分かる
