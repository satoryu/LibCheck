# Issue #11: 登録図書館の管理 - 要件定義

## Problem Statement

ユーザーが Issue #10 で選択した図書館をローカルストレージに永続化し、登録済み図書館の一覧表示・削除ができるようにする。また、BottomNavigationBar を導入してホーム画面と図書館管理画面をタブで切り替えられるようにする。

## Requirements

### Functional Requirements

1. **BottomNavigationBar の実装**: ホーム・図書館・履歴の3タブを持つ BottomNavigationBar を実装する（履歴タブは Phase 4 のためプレースホルダー）
2. **図書館登録**: 図書館一覧画面で選択した図書館をローカルストレージに永続化する
3. **登録図書館の一覧表示**: 登録済み図書館を図書館タブに一覧表示する（図書館名・所在地）
4. **図書館削除**: 登録済み図書館を確認ダイアログ付きで削除できる
5. **Undo 付き SnackBar**: 削除後に「元に戻す」アクション付きの SnackBar を表示する
6. **空状態表示**: 図書館未登録時に適切なメッセージと登録への導線を表示する
7. **図書館追加への導線**: 図書館管理画面の AppBar に「+追加」ボタンを設置する

### Non-Functional Requirements

1. **データ永続性**: アプリ再起動後も登録図書館が維持されること
2. **即時反映**: 登録・削除操作が即座に UI に反映されること

## Constraints

- 既存の `LocalStorageRepository` / `SharedPreferences` を使用する
- go_router の `StatefulShellRoute` で BottomNavigationBar を実装する
- 履歴タブは Phase 4 で実装するためプレースホルダーとする
- `Library` モデルに JSON シリアライズ機能を追加する

## Acceptance Criteria

1. BottomNavigationBar でホーム・図書館・履歴タブが切り替わる
2. 図書館一覧画面の「登録する」ボタンで選択した図書館がローカルストレージに保存される
3. 図書館タブに登録済み図書館が一覧表示される
4. 削除ボタンで確認ダイアログが表示され、確認後に削除される
5. 削除後に Undo 付き SnackBar が表示され、元に戻せる
6. 図書館未登録時に空状態が表示され、登録画面への導線がある
7. アプリ再起動後も登録済み図書館が維持される
8. すべてのテストがパスする

## User Stories

1. **図書館を管理したいユーザー**として、登録した図書館の一覧を見たい。それにより、どの図書館を登録しているか確認できる。
2. **図書館を整理したいユーザー**として、不要な図書館を削除したい。それにより、必要な図書館だけを残せる。
3. **誤操作をしたユーザー**として、削除を取り消したい。それにより、誤って削除した図書館を復元できる。
