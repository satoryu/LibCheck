# Issue #45: ISBN手動入力から検索結果に遷移した際に「別の本をスキャンする」と表示される

## Problem Statement

検索結果ページのボタンが常に「別の本をスキャンする」（カメラアイコン付き）となっているが、ISBN手動入力から遷移した場合にはラベルとアイコンが文脈に合わない。

## Requirements

### Functional Requirements

- バーコードスキャンから遷移した場合: 「別の本をスキャンする」（カメラアイコン）
- ISBN手動入力から遷移した場合: 「別の本を検索する」（検索アイコン）
- 検索履歴から遷移した場合: 「別の本を検索する」（検索アイコン）

### Non-Functional Requirements

- ルーターのクエリパラメータ `source` で遷移元を判別する

## Constraints

- `Navigator.of(context).pop()` による戻る動作自体は変更しない
- 既存のルーティング構造を大きく変更しない

## Acceptance Criteria

1. バーコードスキャンからの遷移時に「別の本をスキャンする」ボタンが表示されること
2. ISBN入力からの遷移時に「別の本を検索する」ボタンが表示されること
3. 検索履歴からの遷移時に「別の本を検索する」ボタンが表示されること
4. いずれのボタンもタップで前のページに戻ること
5. 既存のテストがすべてパスすること

## User Stories

- ユーザーとして、ISBN入力で検索した後に「別の本を検索する」と表示されることで、直前に行った操作と一致した案内を受けたい
