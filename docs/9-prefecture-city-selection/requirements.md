# Issue #9: 都道府県・市区町村選択UI - 要件定義

## Problem Statement

ユーザーが利用する図書館を登録するためには、まず図書館の所在地を都道府県・市区町村で絞り込む必要がある。カーリルAPIの図書館検索エンドポイント（`/library`）は `pref`（都道府県）と `city`（市区町村）をパラメータとして受け取るため、これらを選択するUIが必要。

## Requirements

### Functional Requirements

1. **都道府県一覧表示**: 日本の47都道府県を地方ブロック（北海道・東北、関東、中部、近畿、中国、四国、九州・沖縄）ごとにグループ化して表示する
2. **都道府県検索フィルター**: テキスト入力により都道府県名を部分一致でフィルタリングできる
3. **市区町村一覧表示**: 選択した都道府県に属する市区町村を一覧表示する
4. **市区町村検索フィルター**: テキスト入力により市区町村名を部分一致でフィルタリングできる
5. **画面遷移**: 都道府県選択 → 市区町村選択 → 図書館一覧（次Issue）の順に遷移する
6. **戻るナビゲーション**: 各画面から前の画面に戻れる

### Non-Functional Requirements

1. **オフライン対応**: 都道府県・市区町村データはアプリに静的に埋め込み、ネットワーク不要で表示できること
2. **レスポンス速度**: 一覧表示・フィルタリングは即座に反映されること
3. **アクセシビリティ**: スクリーンリーダー対応、十分なタップ領域（48dp以上）

## Constraints

- 都道府県データは47都道府県固定（静的データ）
- 市区町村データは静的データとしてアプリに埋め込む
- カーリルAPIの `pref`・`city` パラメータに一致する文字列を使用する
- go_router を使用したルーティング

## Acceptance Criteria

1. `/library/add` で都道府県一覧画面が表示される
2. 都道府県が地方ブロックごとにグループ化されている
3. 都道府県名で検索フィルターが機能する
4. 都道府県をタップすると `/library/add/:pref` に遷移し市区町村一覧が表示される
5. 市区町村名で検索フィルターが機能する
6. 市区町村をタップすると `/library/add/:pref/:city` に遷移する（次Issueで図書館一覧を実装）
7. すべてのテストがパスする

## User Stories

1. **図書館を探すユーザー**として、都道府県を選択したい。それにより、利用する図書館の所在地を絞り込める。
2. **図書館を探すユーザー**として、市区町村を選択したい。それにより、具体的な地域の図書館を検索できる。
3. **図書館を探すユーザー**として、都道府県・市区町村を検索したい。それにより、素早く目的の地域を見つけられる。
