# Issue #10: 図書館一覧表示・選択 - 要件定義

## Problem Statement

ユーザーが都道府県・市区町村を選択した後、その地域にある図書館の一覧を表示し、登録したい図書館を選択できるようにする必要がある。カーリルAPIの `/library` エンドポイントから図書館データを取得し、チェックボックスで複数選択できるUIを提供する。

## Requirements

### Functional Requirements

1. **図書館一覧取得**: 選択された都道府県・市区町村に基づき、カーリルAPIから図書館一覧を取得する
2. **図書館一覧表示**: 図書館名・住所を含むリストを表示する
3. **複数選択**: チェックボックスにより複数の図書館を選択できる
4. **選択数表示**: 現在選択中の図書館数を表示する
5. **登録ボタン**: 選択した図書館を登録するボタンを提供する（実際の永続化は次Issue）
6. **ローディング表示**: API通信中にローディングインジケータを表示する
7. **エラー表示**: API通信エラー時にエラーメッセージと再試行ボタンを表示する

### Non-Functional Requirements

1. **レスポンス速度**: API取得後の一覧表示は即座に行われること
2. **アクセシビリティ**: チェックボックスのタップ領域が十分であること

## Constraints

- 既存の `LibraryRepository.getLibraries(pref, city)` を使用する
- go_router の既存ルート `/library/add/:pref/:city` のプレースホルダーを置き換える
- 選択結果は次の Issue #11 でローカルストレージに永続化する

## Acceptance Criteria

1. `/library/add/:pref/:city` で図書館一覧画面が表示される
2. カーリルAPIから取得した図書館が一覧表示される
3. チェックボックスで複数の図書館を選択・解除できる
4. 選択中の図書館数が表示される
5. API通信中にローディング表示がある
6. API通信エラー時にエラー表示と再試行ボタンがある
7. すべてのテストがパスする

## User Stories

1. **図書館を登録したいユーザー**として、地域の図書館一覧を見たい。それにより、利用する図書館を選べる。
2. **複数の図書館を利用するユーザー**として、複数の図書館を一度に選択したい。それにより、効率的に登録できる。
