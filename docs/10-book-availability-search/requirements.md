# Issue #10: 蔵書検索・結果表示 - 要件定義

## Problem Statement

ユーザーがISBN（バーコードスキャンまたは手動入力）で取得した書籍について、登録済み図書館での蔵書有無・貸出状況を確認できる画面がない。カーリルAPIの蔵書検索エンドポイントは実装済みだが、UIが未実装のため利用できない状態。

## Requirements

### Functional Requirements

1. ISBNをパスパラメータとして受け取り、蔵書検索結果を表示する
2. 登録済み図書館のsystemIdを自動取得して検索に使用する
3. カーリルAPIのポーリング中は図書館ごとにローディング状態を表示する
4. 図書館ごとの蔵書状態（貸出可能、貸出中、蔵書なし等）をセマンティックカラーで表示する
5. 予約URLがある場合はリンクを表示する
6. 「別の本をスキャンする」ボタンで再スキャンへ遷移できる
7. 図書館が未登録の場合は登録を促すメッセージを表示する

### Non-Functional Requirements

1. ポーリング中もUIがブロッキングされない
2. 結果が返った図書館から順次表示される（プログレッシブ表示）

## Constraints

- カーリルAPIはポーリング方式（`continue=1` の間はセッションで再リクエスト）
- 既存の `LibraryRepository.checkBookAvailability` と `CalilApiClient.checkAvailability` を利用する
- ルートは `/result/:isbn` とする（design-guidelines 準拠）

## Acceptance Criteria

1. `/result/:isbn` に遷移すると蔵書検索が自動実行される
2. ローディング中に `CircularProgressIndicator` が表示される
3. 登録図書館ごとに蔵書状態がカードで表示される
4. 貸出可能（緑）、貸出中（オレンジ）、蔵書なし（グレー）の色分けが正しい
5. エラー時にエラーメッセージと再試行ボタンが表示される
6. 図書館未登録時に登録誘導メッセージが表示される
7. 全テストがパスする

## User Stories

1. ユーザーとして、ISBNを指定して登録図書館の蔵書状況を確認したい
2. ユーザーとして、検索中も進捗を視覚的に確認したい
3. ユーザーとして、貸出可能な図書館をすぐに見分けたい
4. ユーザーとして、検索結果から別の本の検索に素早く移りたい
