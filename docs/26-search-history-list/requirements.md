# Issue #26: 検索履歴一覧・再検索 — 要件定義

## Problem Statement

Issue #25 で検索履歴の保存基盤を構築したが、ユーザーが保存された履歴を閲覧・操作する UI がない。現在は `HistoryPlaceholderPage` がプレースホルダーとして表示されている。この画面を実際の検索履歴一覧画面に置き換え、過去の検索結果を参照したり、再検索を行えるようにする。

## Requirements

### Functional Requirements

1. **履歴一覧表示**: 保存された検索履歴を新しい順にリスト表示する
2. **履歴カード表示**: 各履歴には以下の情報を表示する
   - ISBN
   - 検索日時（相対表示: 「今日」「昨日」「○日前」等、または日付フォーマット）
   - 蔵書状況のサマリー（最も良い状態をバッジで表示）
3. **再検索**: 履歴カードをタップすると、そのISBNで蔵書検索結果画面（`BookSearchResultPage`）に遷移して再検索を実行する
4. **個別削除**: スワイプまたはメニューから個別の履歴を削除できる
5. **全件削除**: 画面のアクションから全履歴を一括削除できる
6. **空状態**: 履歴が0件の場合はメッセージを表示する
7. **プレースホルダー置換**: 既存の `HistoryPlaceholderPage` を新しい画面に置き換える

### Non-Functional Requirements

1. **レスポンシブ**: 履歴リストはスムーズにスクロールすること
2. **一貫性**: 既存のデザインガイドラインに従ったUIデザイン

## Constraints

- 既存の `SearchHistoryProvider`（Issue #25）を使用する
- 既存のルーティング構成（`/history` パス）をそのまま活用する
- BottomNavigationBar の「履歴」タブは変更不要

## Acceptance Criteria

1. 「履歴」タブに検索履歴一覧が表示される
2. 各履歴カードに ISBN、検索日時、蔵書状況サマリーが表示される
3. 履歴カードタップで蔵書検索結果画面に遷移し再検索が実行される
4. スワイプで個別削除ができる
5. 全件削除ボタンで全履歴を削除できる（確認ダイアログ付き）
6. 履歴0件時に空状態メッセージが表示される
7. すべてのウィジェットにテストが存在する

## User Stories

1. **履歴の閲覧**: ユーザーとして、過去に検索した書籍の一覧を確認できるので、以前調べた本の蔵書状況を思い出せる
2. **再検索**: ユーザーとして、履歴から再検索できるので、蔵書状況の変化を確認できる
3. **履歴の整理**: ユーザーとして、不要な履歴を削除できるので、履歴リストを見やすく保てる
