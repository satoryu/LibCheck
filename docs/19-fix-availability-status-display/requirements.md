# 要件定義: 蔵書状況の表示バグ修正 (#19)

## Problem Statement

蔵書検索結果で、すべての図書館が「貸出可能」と表示されるバグがある。

カーリルAPIは分館（libKey）ごとのステータスを返すが、`LibraryAvailabilityCard`がシステム全体で最も優先度の高いステータス（`LibraryStatus.status`）を表示に使用しているため、1つでも「貸出可」の分館があると全分館が「貸出可能」と表示されてしまう。

## Requirements

### Functional Requirements

- FR-1: 各図書館カードは、その図書館のlibKeyに対応する個別の蔵書ステータスを表示すること
- FR-2: libKeyに対応するステータスが存在しない場合は、`notFound`を表示すること
- FR-3: 既存のシステムレベル集約ステータス（`LibraryStatus.status`）は後方互換性のため保持すること

### Non-Functional Requirements

- NFR-1: 既存のテストが引き続きパスすること
- NFR-2: 新機能に対するユニットテストとウィジェットテストを追加すること

## Constraints

- カーリルAPIのレスポンス形式に準拠すること
- Flutter/Dartの標準的なコーディング規約に従うこと

## Acceptance Criteria

- AC-1: 蔵書検索結果で、各図書館カードがその分館固有のステータスを正しく表示する
- AC-2: 「貸出可」の分館と「貸出中」の分館が混在する場合、それぞれ異なるステータスが表示される
- AC-3: libKeyに対応するステータスが存在しない場合、「蔵書なし」が表示される
- AC-4: すべての既存テストがパスする
- AC-5: 新しいメソッドとウィジェットの動作に対するテストが追加されている

## User Stories

- US-1: ユーザーとして、蔵書検索結果で各図書館の正確な蔵書状況を確認したい。それにより、実際に本が借りられる図書館を正しく判断できる。
